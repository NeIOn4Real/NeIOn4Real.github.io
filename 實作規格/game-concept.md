# VentureTown（創業小鎮）— 遊戲概念文件

> **Status**: Approved | **Created**: 2026-04-11 | **Last Updated**: 2026-04-20

## Elevator Pitch

在 4x4 的地獄小鎮格子上排列設施，將 1 塊錢的資源投入後沿路經過設施轉換增值，用 10 回合達成收益目標即可過關。結合路徑規劃、設施組合和合夥人協同的輕量回合制策略遊戲。

## 核心幻想（Core Fantasy）

玩家扮演被扔到地獄的創業者，在兩位惡魔少女（譚雅和蕾雅）的「幫助」下，用有限的設施和資源打造出一條黃金生產線。每一輪都是一次小型經濟引擎的設計挑戰——用最少的回合、最巧的路徑，壓榨出最大的收益。

---

## 獨特賣點（Unique Hook）

- **路徑即策略**：資源從四邊投入後自動沿直線行進，玩家透過設施排列和物流方向改變路徑，同一張地圖不同投入方向產生截然不同的收益
- **角色陪伴感**：兩位有性格的角色提供 114+ 條情境台詞，不是冷冰冰的 UI 而是有溫度的互動
- **稀有度成長**：隨輪數推進，更強力的 R/SR/SSR 設施和合夥人逐步解鎖，每輪都有新的策略組合可能

## 玩家體驗分析（MDA 框架）

### 目標美學（玩家感受）

| 優先順序 | 美學 | 實現方式 |
|----------|------|----------|
| 1 | **挑戰（Challenge）** | 動態難度系統讓每輪都有適當壓力 |
| 2 | **發現（Discovery）** | 新設施/合夥人/事件的組合不斷湧現 |
| 3 | **表達（Expression）** | 多流派 Build（人力/物流/貿易/拆遷/電子）允許個人風格 |
| 4 | **陪伴（Fellowship）** | 角色台詞和立繪營造「夥伴在身邊」的感覺 |

### 核心動態（湧現行為）

| 動態 | 觸發機制 |
|------|----------|
| 路徑最佳化 | 玩家反覆嘗試不同投入方向，尋找最高收益路徑 |
| 流派構築 | 詞條加權系統讓玩家越深入某流派，越容易取得相關設施 |
| 風險評估 | 隨機事件預告讓玩家在「避險」和「承受」之間抉擇 |
| 資源抉擇 | 行動費用 vs 保留收益的 trade-off |

### 核心機制（系統）

| 機制 | 說明 |
|------|------|
| 4x4 格子放置 | 設施排列和路徑規劃 |
| 資源投入 | 四方向投入，沿直線行進觸發設施效果 |
| 類型轉換鏈 | 金錢 → 原料 → 商品 → 金錢 的轉換增值 |
| 合夥人協同 | 56 位合夥人提供被動/主動效果 |
| 隨機事件 | 19 種事件製造變數和機會 |
| 動態難度 | 自動追蹤玩家表現調整目標 |
| 稀有度解鎖 | 輪數推進解鎖更強力的選項 |

## 玩家動機分析

### 心理需求（自我決定論）

| 需求 | 滿足方式 |
|------|----------|
| **自主性** | 每回合自由選擇：放置、投入方向、行動選項、合夥人使用 |
| **勝任感** | 動態難度確保不會太簡單或太挫折；過關回饋明確 |
| **歸屬感** | 角色台詞互動、合夥人「夥伴」關係 |

### 玩家類型吸引力（Bartle 分類）

| 類型 | 吸引力 | 設計支撐 |
|------|--------|----------|
| **成就者** | 高 | 輪數推進、目標達成、稀有度收集 |
| **探索者** | 高 | 79 設施 × 56 合夥人的組合空間 |
| **社交者** | 中 | 角色陪伴感（無多人模式） |
| **競爭者** | 中低 | 無排行榜（未來可加） |

### 心流設計

| 要素 | 實現 |
|------|------|
| 清晰目標 | Header 顯示目標/收益/進度條 |
| 即時回饋 | 資源行進動畫 + 收益飛行數字 + 音效 |
| 挑戰/技能平衡 | `difficultyMult` 0.5~3.0 自動調整 |
| 控制感 | 可預覽事件、可重排設施、可存讀檔 |

## 核心迴圈

### 即時（每次投入，~10 秒）

```
選方向 → 資源沿路行進 → 設施逐個觸發 → 收益結算 → 合夥人效果
```

### 短期（每回合，~1 分鐘）

```
行動選項二選一 → 放置/排列設施 → 投入人材 → 投入資源 → 結束回合
```

### 中期（每輪，~5 分鐘）

```
10 回合 → 累積收益 ≥ 目標 → 過關 → 新目標 + 難度調整 → 下一輪
```

### 長期（跨輪）

```
稀有度解鎖更強設施 → Build 漸趨成熟 → 挑戰更高目標 → 流派定型
```

### 留存鉤子

- 「再一輪」慾望：過關後新目標只高一點，感覺「下一輪一定能過」
- 稀有度驚喜：偶爾在低輪數抽到 SSR 帶來興奮感
- 角色羈絆：台詞系統讓玩家對角色產生情感連結
- 流派深化：詞條系統獎勵深耕特定流派

## 設計支柱

### 支柱 1：簡單規則，深度組合

> 每個設施的效果一句話就能說清，但 79 個設施和 56 個合夥人的組合產生無窮策略深度。

**設計測試**：如果一個新設施需要超過兩行描述才能說清效果，就該拆分或簡化。

### 支柱 2：每輪都是新挑戰

> 動態難度 + 隨機事件 + 稀有度解鎖確保沒有兩輪完全相同的體驗。

**設計測試**：如果玩家可以用完全相同的策略連過 5 輪而不需調整，難度系統需要加壓。

### 支柱 3：角色不是裝飾

> 台詞系統和合夥人機制讓角色成為遊戲體驗的核心部分，而非可跳過的旁白。

**設計測試**：如果移除角色台詞後遊戲體驗幾乎不變，需要加強角色與機制的綁定。

### 反支柱（Anti-Pillars）

- **這不是放置遊戲**：沒有離線收益，每次投入都需要玩家主動決策
- **這不是 Roguelike**：沒有永久死亡，失敗可重新開始且難度降低
- **這不是多人遊戲**：專注單人體驗，不設計排行榜或 PvP

## 技術考量

| 面向 | 決策 |
|------|------|
| 單檔架構 | 降低部署複雜度，但影響維護性（8300+ 行） |
| 無框架 | 零依賴，載入快，但 DOM 操作較冗長 |
| localStorage | 簡單可靠，但有 5MB 限制 |
| Web Audio API | 免外部音檔，但合成音效表現力有限 |
| CSS 動畫 | 效能好，但複雜動畫控制有限（setTimeout 遞迴） |

## 風險與開放問題

### 設計風險

| 風險 | 嚴重度 | 緩解措施 |
|------|--------|----------|
| 後期收益爆炸 | 高 | 動態難度 + 大熱波懲罰機制 |
| 設施/合夥人過多難以平衡 | 高 | 稀有度分層 + 詞條加權控制出現頻率 |
| 新手資訊過載 | 中 | 9 步教學 + 稀有度漸進解鎖 + 安全網機制 |
| 流派不平衡 | 中 | 詞條系統自然引導 + 待持續 playtest |

### 技術風險

| 風險 | 嚴重度 | 緩解措施 |
|------|--------|----------|
| 單檔維護困難 | 高 | 模組化調度表（FACILITY_FX / PARTNERS hook） |
| 動畫 setTimeout 殘留 | 中 | `_stepAnimGen` 世代計數器 + `cancelStepAnim()` |
| 存檔相容性 | 中 | 版本號 + 反序列化安全清理 |
