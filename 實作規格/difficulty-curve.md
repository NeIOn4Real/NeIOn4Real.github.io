## 難度哲學

**Accessible（可親近型）**

VentureTown 追求讓所有玩家都能體驗核心樂趣，同時為高手提供足夠的挑戰深度。難度不來自懲罰，而是來自目標的自然成長——玩家永遠在「剛好夠用」和「可以更好」之間尋找最佳解。

**核心原則：**
- 第 1 輪任何人都能過關（教學保護 + 低目標）
- 系統追蹤玩家表現，自動調整到「剛好有挑戰」的甜蜜點
- 失敗不懲罰：重開時難度重置
- 高手不無聊：動態加壓 + 大熱波防刷分

## 難度軸

| 軸 | 1（最易） | 5（標準） | 10（最難） | 玩家控制 |
|----|----------|----------|-----------|---------|
| **目標壓力** | 目標 = 收益的 30% | 目標 = 收益的 60% | 目標 = 收益的 90%+ | 無（系統自動） |
| **路徑複雜度** | 3 設施直線 | 物流轉向 + 5 設施 | 全格子滿配 + 疊加 | 玩家放置 |
| **資訊量** | N 級設施（效果直覺） | R+SR 混合（需思考組合） | SSR+合夥人交互（需計算） | 稀有度漸進 |
| **隨機干擾** | 平靜的一天 | 行業熱潮 / 區域效應 | 莫菲定律 / 颱風 | 可更換事件 |
| **資源管理** | 免費行動多 | 費用與收益平衡 | 每筆費用都要精算 | 行動選擇 |

## 入門坡度（Onboarding Ramp）

### 各階段玩家知識

| 階段 | 玩家已知 | 玩家將學 |
|------|----------|----------|
| 教學開始 | 無 | 設施放置、資源投入方向 |
| 教學中段 | 基本放置 | 設施轉換鏈（金→原→商→金） |
| 教學結束 | 轉換鏈 | 結束回合、事件預告、收益目標 |
| 第 1 輪 | 核心規則 | 行動選項（購買/招募）、合夥人效果 |
| 第 2 輪 | 行動系統 | 路徑優化、人材投入、常駐商店 |
| 第 3 輪 | 路徑規劃 | SR 設施特效、物流轉向、流派選擇 |

### 機制引入順序

| 順序 | 機制 | 引入時機 | 方式 |
|------|------|----------|------|
| 1 | 設施放置 | 教學 Step 1 | 強制引導 |
| 2 | 資源投入 | 教學 Step 3 | 強制引導 |
| 3 | 轉換鏈 | 教學 Step 4~5 | 強制體驗 |
| 4 | 結束回合 | 教學 Step 6 | 按鈕引導 |
| 5 | 收益目標 | 教學 Step 9 → 第 1 輪 | 自然體驗 |
| 6 | 行動選項 | 第 1 輪回合 2 | 彈出選擇器 |
| 7 | 合夥人 | 第 1 輪起始選擇 | 3 選 1 |

| 9 | 人材系統 | 取得人力設施/合夥人時 | 左面板出現 |

| 11 | 常駐商店 | 第 1 輪起 | 右面板按鈕 |

| 14 | SSR 設施 | 第 5 輪起明顯 | 稀有度加權 |

### 第一次失敗

**預期時機**：第 3~4 輪（目標 58~89，需要 R 級設施但可能還沒找到好的組合）

**設計意圖**：讓玩家理解「需要優化路徑和組合」而非「多玩幾回合就好」

**緩解措施**：
- 失敗時 `difficultyMult` 重置為 1.0
- 重開後顯示起始合夥人選擇（可選不同策略）
- 安全網機制在連續零收益時介入

### 玩家首次感到掌控

**預期時機**：第 2 輪末（約第 20 回合）

**觸發條件**：玩家首次用 4+ 設施路徑達成 10+ 收益，發出「原來這樣組合更好」的理解

## 難度波峰與波谷

| 事件 | 難度變化 | 設計意圖 |
|------|----------|----------|
| 設施補給事件 | ↓ 暫時降低 | 喘息 + 獎勵感 |
| 莫菲定律 | ↑↑ 暫時飆升 | 打亂固化路徑，強迫重新思考 |
| 颱風 | ↑ 中度上升 | 限制方向 + 物流失效，需要適應 |
| 地區叛亂 | ↑ 中度上升 | 消滅角落設施，需要重建 |
| 就業輔助 | ↓ 暫時降低 | 安全網機制的顯性表現 |
| 蕾雅的禮物 | ↓ 暫時降低 | 設施永久加成 |
| SSR 首次出現 | ↓↓ 明顯降低 | Power spike，玩家感到強大 |
| 目標跳升（新輪） | ↑ 自然上升 | 保持挑戰感 |

## 平衡槓桿（Balancing Levers）

| 槓桿 | 目前值 | 調整範圍 | 影響 |
|------|--------|----------|------|
| `初始目標` | 20 | 10~40 | 第 1 輪難度 |
| `目標成長倍率` | 1.4 | 1.2~1.8 | 後期目標攀升速度 |
| `目標固定加值` | 8 | 0~20 | 低輪目標的底線 |
| `difficultyMult 範圍` | 0.5~3.0 | 0.3~4.0 | 自動調整的彈性 |
| `difficultyMult 初始` | 1.0 | — | 固定 |
| `速通加壓（≤3回合）` | +20% | 10%~30% | 對強者的加壓強度 |
| `掙扎降壓（>9回合）` | -10% | 5%~15% | 對弱者的保護強度 |
| `稀有度解鎖輪數` | R@2, SR@3, SSR@5 | 可個別調 | 設施/合夥人可用時機 |
| `行動費用基礎` | 3~8 | — | 收益消耗速度 |
| `費用成長率` | +35%/輪 | 20%~50% | 後期行動價值 |
| `安全網觸發` | 3 回合零收益 | 2~5 | 保護介入時機 |
| `大熱波觸發` | 收益 ≥100× 目標 | 50~200 | 防刷分閾值 |

## 動態難度系統

### difficultyMult 調整規則

```
每輪過關時計算 delta：

速度調整：
  ≤3 回合達標 → +20%
  ≤5 回合達標 → +10%
  ≤7 回合達標 → +3%
  ≤9 回合達標 → ±0%
  >9 回合達標 → -10%

連續速通：
  連續 2 輪 ≤3 回合 → 額外 +8%
  連續 3 輪 ≤3 回合 → 額外 +15%

超額比率：
  收益 ≥10× 目標 → +30%
  收益 ≥5× 目標 → +20%
  收益 ≥3× 目標 → +12%
  收益 ≥2× 目標 → +6%

平均超額（近 3 輪）：
  平均 ≥5× → +15%
  平均 ≥3× → +8%

掙扎保護：
  超額 <1.3× → -5%
  連續多輪 <1.5× → -5%

最終：difficultyMult = clamp(difficultyMult + delta, 0.5, 3.0)
```

### 目標公式

```
新目標 = round(舊目標 × 1.4 × difficultyMult + 8)
```

### 典型 difficultyMult 軌跡

| 玩家類型 | 第 1 輪 | 第 3 輪 | 第 5 輪 | 第 7 輪 |
|----------|---------|---------|---------|---------|
| 新手 | 1.0 | 0.85 | 0.7 | 0.6 |
| 普通 | 1.0 | 1.05 | 1.1 | 1.15 |
| 高手 | 1.0 | 1.35 | 1.8 | 2.3 |
| 刷分者 | 1.0 | 1.6 | 2.5 | 3.0（上限） |

## 跨系統難度交互

| 系統 A | 系統 B | 交互方式 |
|--------|--------|----------|
| 稀有度解鎖 | 目標公式 | R/SR 設施讓收益跳升，但目標同步成長 |
| 合夥人負面效果 | 動態難度 | 負面效果增加失敗風險 → 降壓 |
| 隨機事件 | 玩家路徑 | 莫菲定律打亂最佳路徑 → 臨時難度飆升 |
| 大熱波 | 動態難度 | 極端收益觸發大熱波 → 強制降速 |
| 安全網 | 稀有度 | 安全網保證物流中心和核心合夥人出現 |
| 詞條加權 | 流派深度 | 深耕單一流派提高同流派出現率 → 降低組合難度 |

## 無障礙考量

### 可調整

| 機制 | 效果 | 實現方式 |
|------|------|----------|
| 動態難度 | 自動適應玩家水平 | `difficultyMult` 自動調整 |
| 安全網 | 保護掙扎的玩家 | 靜默提供核心設施/合夥人 |
| 失敗重置 | 不累積挫折感 | `difficultyMult` 回到 1.0 |
| 教學可跳過 | 老手不需重看 | 標題畫面選新遊戲 |
| 存讀檔 | 隨時中斷/繼續 | localStorage + JSON 匯出 |

### 不可調整（設計核心）

| 限制 | 原因 |
|------|------|
| 10 回合限制 | 核心壓力來源，移除後失去策略深度 |
| 資源起始值 1 | 確保每輪從零開始構建，不依賴累積 |
| 事件隨機性 | 製造變數是核心體驗，可透過「更換事件」緩解 |

## 驗證清單

### 入門（教學~第 2 輪）

- [ ] 教學 9 步可在 5 分鐘內完成
- [ ] 第 1 輪目標（20）在 4 回合內可達
- [ ] 第 1 輪不出現 SR/SSR 設施（極低機率例外）
- [ ] 安全網在第 1 輪連續零收益時觸發
- [ ] 角色台詞至少觸發 5 次

### 早期（第 3~4 輪）

- [ ] 玩家開始有意識地規劃路徑（非隨意放置）
- [ ] R 級設施明顯出現，帶來收益跳升感
- [ ] 第一次失敗（如果發生）不會讓玩家放棄
- [ ] 重開後 difficultyMult 重置有效

### 中期（第 5~6 輪）

- [ ] SR 設施成為核心，Build 開始定型
- [ ] 合夥人組合產生明顯策略差異
- [ ] 目標壓力足夠（不能每輪第 2 回合就達標）
- [ ] difficultyMult 對高手有效加壓

### 後期（第 7+ 輪）

- [ ] SSR 設施帶來新的策略層次
- [ ] 目標攀升速度不超過收益成長速度
- [ ] 大熱波在極端情況下正確觸發
- [ ] 連續通關 10+ 輪仍有挑戰感

### 無障礙

- [ ] 所有關鍵操作支援拖曳和點擊兩種方式
- [ ] 數值變化有視覺回饋（badge + 飛行動畫）
- [ ] 音效非必要（關閉音效不影響遊玩）
- [ ] 格子懸停 tooltip 提供完整資訊

## 開放問題

- [ ] difficultyMult 上限 3.0 是否足夠？（極端高手可能需要更高）
- [ ] 安全網靜默介入是否讓部分玩家感到「被操控」？
- [ ] 大熱波觸發閾值（100× 目標）是否太高？需要更多 playtest 數據
- [ ] 是否需要「自訂難度」選項（固定 difficultyMult）？
- [ ] 輪數 10+ 後是否需要新的目標公式（避免數值過大）？
